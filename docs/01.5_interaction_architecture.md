# Purpose

This document defines how Project Cortex behaves.

The reference artwork defines appearance.

This document defines interaction.

Every animation, transition, highlight and response should originate from this specification.

If the project feels intuitive, responsive and satisfying to use, this document has succeeded.

---

# Philosophy

The user is not operating software.

The user is exploring an engineer's mind.

Every interaction represents the discovery of knowledge.

Nothing should behave like a dashboard.

Nothing should behave like a game.

Nothing should behave like a slideshow.

Every interaction should feel like revealing another part of an engineered system.

# Interaction States

Every lobe exists in one of four states.

Idle

Hovered

Selected

Focused

Only visual intensity changes.

Geometry should never distort dramatically.

---

# Idle State

Brain performs passive computation.

Characteristics

Slow electrical movement

Gentle breathing glow

Minimal bloom

Subtle particle motion

Very slow floating motion

Nothing demands attention.

---

# Hover State

Hover communicates possibility.

Changes

Increase emissive intensity

Increase packet frequency

Slight rim light boost

Cursor feedback

Connector node illuminates

No panels appear.

Hover should never feel overwhelming.

---

# Selection State

Selection means

"I want to inspect this knowledge."

Sequence

1

Click detected.

↓

2

Lobe brightens.

↓

3

Electrical packets accelerate.

↓

4

Energy travels toward connector origin.

↓

5

Connector line grows.

↓

6

Knowledge panel materializes.

↓

7

Text fades upward.

↓

8

Brain returns to normal activity.

Total duration

Approximately

600–900 ms

The interaction should feel deliberate.

Never abrupt.

---

# Deselection

Reverse the process.

Text fades.

↓

Panel retracts.

↓

Connector retracts.

↓

Electrical intensity returns to idle.

↓

Brain resumes passive computation.

---

# Knowledge Panels

Panels never "pop."

Panels deploy.

Think of precision instrumentation.

Characteristics

Soft scaling

Opacity fade

Minor upward translation

Glass material

Thin illuminated border

Subtle blur

Never bounce.

Never overshoot.

---

# Panel Placement

The brain remains the visual centerpiece.

Panels should never obscure the brain.

When only one panel exists

Place it in the lower-left interaction zone.

When multiple panels exist

Continue clockwise around the viewport.

Example

1 Panel

Lower Left

2 Panels

Lower Left

Left Center

3 Panels

Lower Left

Left Center

Upper Left

4 Panels

Continue around the viewport.

Eventually

Upper Right

Right Center

Lower Right

This creates a circular ring of information around the brain.

The center always remains clear.

# Why the Temporal Lobes Are Split

The anatomical temporal lobe exists on both hemispheres.

Project Cortex intentionally maps them independently.

Left Temporal

Analytical computation

Core engineering theory

Right Temporal

Applied intelligence

AI systems

Prompt engineering

This improves usability while remaining anatomically plausible.

# Camera Behaviour

Idle

Very slow orbit.

Hover

Tiny movement toward hovered lobe.

Selection

Camera eases closer.

Approximately

5–10%

Never dramatic zoom.

Deselection

Return smoothly.

The camera should never make users feel lost.

# Connector Lines

Connector lines represent knowledge pathways.

They do not exist until required.

Animation

Node glows.

↓

Line grows.

↓

Electrical packet travels.

↓

Panel appears.

↓

Line remains softly illuminated.

When deselected

Reverse.

---

# Electrical Behaviour

Each lobe owns an independent electrical network.

Packets remain inside their own region.

No colour bleeding.

No packet crossing.

No emissive leakage.

This reinforces the illusion that each knowledge family is its own subsystem.

---

# Multi Selection

Users may select multiple lobes.

Each selection behaves independently.

Panels remain visible.

Connector lines remain active.

Electrical routing continues simultaneously.

The brain should feel more alive as more regions become active.

---

# Hover Priority

Hover never overrides selection.

Selected lobes always remain active.

Hovered unselected lobes temporarily increase activity.

# Performance Rules

Animations should remain GPU friendly.

Avoid layout thrashing.

Avoid unnecessary React re-renders.

Use shader uniforms whenever possible.

Prefer interpolation over abrupt value changes.

---

# Emotional Goal

The interaction should never feel flashy.

It should feel

confident

precise

engineered

and quietly intelligent.

Users should feel like they are uncovering knowledge rather than opening menus.

---

# Success Criteria

The interaction succeeds when visitors instinctively understand:

"This brain is not decorative.

It is the navigation system for the engineer's knowledge."

Every animation should reinforce that idea.
