# 1. Executive Summary

Project Cortex is the flagship interactive visualization of my portfolio.

It is not a decorative 3D model.

It is not a hero animation.

It is not a neural network visualization.

It is a digital representation of how I think as an engineer.

The centerpiece of the portfolio is a cybernetic brain that visually communicates my technical abilities through a living electronic system.

The experience should immediately communicate three things:

• Engineering
• Intelligence
• Precision

before the visitor even begins reading.

The brain should feel like a sophisticated technological artifact rather than a biological object.

Visitors should instinctively understand that every illuminated region represents a real technical capability that can be explored.

The component serves simultaneously as

- portfolio navigation
- technical demonstration
- visual storytelling
- interaction showcase
- graphics showcase
- engineering showcase

This single component should become the most memorable part of the entire website.

---

# 2. Design Philosophy

The project is guided by one principle.

> Intelligence made tangible.

Instead of showing skills inside boring cards or progress bars, the skills become physical regions inside an engineered brain.

The user explores my abilities by exploring the brain itself.

The portfolio should feel less like a website and more like interacting with a futuristic operating system.

Every animation must have purpose.

Every glow must represent information.

Every movement should reinforce the illusion that this is an active computational system.

Nothing should exist purely for decoration.

---

# 3. Core Vision

The brain is NOT:

- medical visualization
- educational model
- anatomy viewer
- neural simulation
- sci-fi prop
- glowing toy
- random cyberpunk object

The brain IS:

A multilayer cybernetic processor.

Imagine a human brain rebuilt by electrical engineers.

The biological tissue has been replaced by engineered materials while preserving the recognizable anatomy.

The object should look manufactured.

It should appear expensive.

It should appear precise.

It should appear computational.

# 7. Rendering Philosophy

The renderer should behave like a miniature graphics engine.

The brain should not be represented using one material.

Instead it is composed of many rendering systems.

Examples include

Base metallic shell

PCB substrate

Copper routing

Fiber optic channels

Embedded electronics

Mechanical joints

Emissive circuitry

Animated electrical packets

Bloom

Volumetric atmosphere

Reflections

Micro surface variation

Each rendering system exists independently.

# 11. Mesh Philosophy

The imported GLB represents anatomy.

Project Cortex represents engineering.

The mesh should evolve into something that appears manufactured.

Additional geometric detail is encouraged.

Possible additions

Raised traces

Engraved channels

Mechanical seams

Fasteners

Circuit housings

Protective shells

Fiber optic conduits

Layered surfaces

Microstructures

The silhouette should remain recognizably human.

Everything else can evolve.

---

# 12. Material Philosophy

No region should appear painted.

No lobe should appear filled.

Color should emerge naturally from engineered components.

The majority of the visible surface should remain dark.

Emissive energy exists only inside conductive pathways.

The material hierarchy should reinforce physical realism.

---

# 13. Animation Philosophy

Movement represents computation.

The brain never behaves like liquid.

It never pulses randomly.

Instead

data travels.

Signals propagate.

Subsystems activate.

Energy transfers.

Information flows.

Animations should imply that work is continuously occurring inside the system.
