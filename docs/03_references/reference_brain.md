Purpose

This document serves as the authoritative visual reference for the Brain component used throughout Project Cortex.

It does not define implementation.

It defines appearance.

Every future rendering decision should preserve the characteristics documented here.

---

# Identity

The brain is the centerpiece of the portfolio.

It is not decoration.

It is not a medical model.

It is the visual representation of an engineer's accumulated knowledge.

The visitor should immediately recognize

Engineering

before

Biology.

---

# Source Asset

Primary Mesh

Brain_port.glb

The GLB provides

- Overall anatomy
- Surface topology
- Major folds
- Brainstem
- Cerebellum

The mesh should never be considered the finished asset.

It is the structural foundation.

# Silhouette

Preserve

- Overall brain proportions
- Left hemisphere
- Right hemisphere
- Brainstem
- Cerebellum
- Natural asymmetry

Do Not Preserve

- Plain biological appearance
- Smooth organic surface
- Medical realism
- Visible tissue appearance

The silhouette remains biological.

The surface becomes engineered.

---

# Visual Goal

Imagine a real human brain rebuilt by electrical engineers.

Every visible square centimeter should appear intentionally manufactured.

The final object should resemble

- Precision electronics
- Industrial design
- Advanced computing hardware

rather than

- Biology
- Plastic
- Medical illustration

---

# Surface Philosophy

The surface should never appear flat.

Every area should contain meaningful detail.

Large uninterrupted regions should be avoided.

Visual density should increase naturally as the viewer zooms closer.

The brain should reward inspection.

---

# Surface Features

The surface may include

- Raised PCB traces
- Recessed routing channels
- Mechanical seams
- Copper routing
- Surface-mounted components
- Integrated circuits
- Microvias
- Contact pads
- Fiber-optic channels
- Metallic reinforcement
- Structural ribs

All additions should follow the existing anatomical curvature.

---

# Circuit Density

High.

The routing should appear intentional rather than decorative.

Different areas should contain different routing densities.

Primary pathways should be wider.

Secondary pathways should branch naturally.

Micro routing should exist between larger structures.

Avoid random decorative line patterns.

---

# Region Separation

Each knowledge region represents an independent subsystem.

Each region should have

- Independent routing
- Independent emissive color
- Independent packet animation

No uncontrolled emissive bleeding.

Boundaries should remain visually clear while still feeling integrated.

# Detail Hierarchy

The eye should discover information in layers.

From far away

Recognize the brain.

Medium distance

Notice the circuitry.

Closer

Notice routing.

Closer

Notice chips.

Closer

Notice vias.

Closer

Notice mechanical detailing.

There should always be another level of detail.

---

# Mechanical Language

The engineering language should remain consistent.

Preferred

- Machined
- Layered
- Precise
- Functional
- Manufactured
- Modular

Avoid

- Organic blobs
- Random sci-fi shapes
- Decorative greebles
- Excessive noise

Every detail should appear engineered.

---

# Brainstem

The brainstem should feel like the central backbone of the system.

It should visually resemble the primary communication bus connecting all knowledge regions.

It may contain denser routing and structural reinforcement than the surrounding anatomy.

---

# Cerebellum

The cerebellum should contain finer routing than the hemispheres.

It represents coordination and infrastructure.

The routing should feel organized and highly interconnected.

---

# Mesh Quality Expectations

Preferred

- Clean topology
- Smooth shading
- Proper normals
- Accurate tangents
- High-resolution silhouette
- Consistent curvature

Avoid

- Visible faceting
- Sharp shading artifacts
- Mesh tearing
- Overlapping geometry
- Broken normals

---

# Scale

The brain should feel substantial.

It should not appear toy-like.

The supporting base should reinforce the perception that this is a precision instrument rather than a floating prop.

---

# Relationship With The Base

The brain should appear mounted.

Not floating randomly.

The support structure should feel intentionally engineered.

Possible support methods include

- Precision pedestal
- Mechanical dock
- Magnetic levitation ring
- Structural cradle

The base should never compete visually with the brain.

---

# Future Mesh Enhancements

If technically feasible

- Additional beveling
- Increased local detail
- Panel segmentation
- Surface displacement
- Embedded mechanical structures
- Procedural PCB detailing

These enhancements should preserve the original anatomical silhouette.

---

# Common Mistakes

Do Not

- Paint color directly onto the mesh
- Use flat emissive regions
- Cover the surface with random glowing lines
- Make every trace identical
- Remove anatomical readability
- Overcrowd every area equally
- Turn the brain into a generic cyberpunk object

---

# Success Criteria

The mesh succeeds when

- The silhouette is immediately recognizable as a brain.
- The surface is immediately recognizable as engineered.
- The viewer discovers more detail the longer they inspect it.
- The brain remains the undisputed centerpiece of the scene.
- Every enhancement appears intentionally manufactured rather than artistically improvised.
