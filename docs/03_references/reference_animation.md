# Project Cortex
# 03_references/reference_animation.md

Version: 1.0

Purpose

This document defines how Project Cortex moves.

Movement represents computation.

Every animation should communicate that the Engineer's Brain is continuously processing knowledge.

No movement exists purely for decoration.

---

# Animation Philosophy

Motion should feel

- Intelligent
- Precise
- Mechanical
- Purposeful
- Calm
- Responsive

Avoid

- Cartoon motion
- Bouncy motion
- Overly energetic motion
- Random movement
- Idle chaos

The brain should feel alive, not busy.

---

# Animation Layers

Animation exists in independent layers.

1. Idle Motion

2. Electrical Activity

3. User Interaction

4. Camera Motion

5. UI Motion

Each layer operates independently while remaining synchronized.

---

# Idle State

The default state.

Characteristics

- Very slow floating motion
- Extremely subtle rotation
- Gentle computational breathing
- Continuous low-level electrical activity

The brain should never become completely static.

---

# Electrical Activity

Electrical activity represents knowledge processing.

Characteristics

- Small energy packets
- Independent routing networks
- Variable packet speeds
- Variable packet frequency
- Continuous operation

The brain should appear to be thinking.

---

# Electrical Packets

Packets represent information.

Not electricity.

Not plasma.

Not flowing liquid.

Each packet should

- Travel smoothly
- Maintain direction
- Follow routing
- Fade naturally
- Restart elsewhere

Packet movement should feel engineered.

---

# Knowledge Regions

Each lobe owns an independent routing system.

Packets should never cross between inactive regions.

Each selected region becomes more active independently.

No color bleeding.

No packet crossover.

---

# Hover Animation

Hover communicates possibility.

Changes

- Slight emissive increase
- Slight packet speed increase
- Slight rim-light enhancement
- Gentle pulse

Duration

150–250 ms

Hover should feel immediate.

---

# Selection Animation

Sequence

1.

Click detected.

↓

2.

Region activates.

↓

3.

Electrical intensity increases.

↓

4.

Energy travels toward connector origin.

↓

5.

Connector line grows.

↓

6.

Knowledge panel deploys.

↓

7.

Brain returns to active idle.

Target Duration

600–900 ms

---

# Knowledge Activation

Knowledge Activation is the signature interaction of Project Cortex.

Every newly selected knowledge region contributes additional computational activity.

The more knowledge regions activated

↓

The more alive the brain becomes.

One active region

Minimal activity.

Three active regions

Noticeably more active.

Seven active regions

Entire brain operating at maximum visual complexity.

The visitor should feel they are awakening the engineer's knowledge rather than opening interface panels.

---

# Knowledge State

Dormant

↓

Awakened

↓

Expanding

↓

Synchronized

Dormant

Only passive computation.

Awakened

One knowledge family active.

Expanding

Multiple systems active simultaneously.

Synchronized

All knowledge families active.

Entire brain fully operational.

---

# Camera Animation

Camera movement should remain subtle.

Hover

Tiny positional shift.

Selection

Gentle focus movement.

Deselection

Smooth return.

Avoid dramatic zooms.

Avoid sudden rotation.

---

# Connector Animation

Connector lines should never instantly appear.

Sequence

Node activates.

↓

Line grows.

↓

Energy packet travels.

↓

Panel deploys.

↓

Connector remains softly illuminated.

Reverse when deselected.

---

# Knowledge Panel Animation

Panels should

- Fade
- Translate upward slightly
- Scale subtly

Never bounce.

Never overshoot.

Never snap into place.

---

# Particle System

Particles provide environmental depth.

Characteristics

- Very small
- Slow
- Sparse
- Independent

Purpose

Suggest ambient computational energy.

Avoid

- Fireflies
- Snow
- Sparks
- Explosions

---

# Base Platform

The supporting base may contain

- Slow glow
- Minor pulse
- Soft energy ring

The base should never compete with the brain.

---

# Animation Timing

Hover

150–250 ms

Selection

600–900 ms

Camera

500–800 ms

Panel Deployment

350–600 ms

Connector Growth

250–400 ms

Knowledge Activation

800–1200 ms

All timings should overlap naturally.

---

# Animation Curves

Preferred

Ease In-Out

Smoothstep

Cubic

Sine

Avoid

Linear motion

Elastic

Bounce

Overshoot

The interface should feel engineered.

---

# Continuous Motion

Even with no interaction

the following remain active

- Idle float
- Packet movement
- Particle drift
- Micro emissive variation

The scene should never appear frozen.

---

# Performance Guidelines

Prefer

Shader animation

GPU interpolation

Uniform updates

Avoid

Frequent React state updates

Expensive CPU animation

Layout recalculation

Animation should remain GPU friendly.

---

# Common Mistakes

Avoid

- Constant flashing
- Rapid blinking
- Rainbow effects
- Random pulses
- Excessive movement
- Identical packet timing
- Synchronized flashing
- Overactive particles

The motion should communicate confidence.

Not excitement.

---

# Success Criteria

The animation succeeds when

- The brain appears continuously operational.
- User interactions feel deliberate.
- Knowledge activation becomes memorable.
- Motion reinforces engineering rather than spectacle.
- Every animation has a functional purpose.

This document serves as the authoritative animation reference for Project Cortex.
