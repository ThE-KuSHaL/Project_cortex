# Project Cortex
# 03_references/reference_colors.md

Version: 1.0

Purpose

This document defines the complete color language of Project Cortex.

Color is never decorative.

Color represents knowledge, computation, interaction, and energy.

Every color used throughout the project must originate from this document.

---

# Color Philosophy

The Engineer's Brain exists inside darkness.

Darkness represents the unknown.

Color represents knowledge.

Light represents computation.

The structure remains neutral.

Only information emits color.

---

# Global Rules

- Never fill large regions with solid color.
- Color exists primarily inside circuitry.
- The substrate remains neutral.
- The brightest colors represent active computation.
- White is reserved for electrical hotspots.
- Colors should never bleed across knowledge regions.

# Particle Colors

Default

Soft white.

Regional particles

Match active lobe.

Particles should never overpower electrical routing.

---

# Bloom Colors

Bloom inherits the emitting color.

No artificial bloom colors.

Bloom should remain soft and localized.

---

# Background Colors

The background should remain almost monochromatic.

Only subtle hints of blue-gray are permitted.

Never introduce strong background colors.

---

# Interaction States

Idle

Muted regional colors.

---

Hover

Slight increase in saturation and brightness.

---

Selected

Maximum regional intensity.

---

Multiple Selection

Each region maintains its own color identity.

No color mixing.

No additive rainbow effects.

---

# Accessibility

All text must maintain sufficient contrast.

Color should never be the only indicator of interaction.

Selection should also modify

- Brightness
- Motion
- Connector state
- Panel visibility

---

# Color Harmony

The complete palette should feel cohesive.

Avoid introducing additional colors unless absolutely necessary.

Future knowledge regions should derive from the existing palette.

---

# Common Mistakes

Avoid

- Rainbow gradients
- Neon overload
- Fully colored lobes
- Random glow colors
- Excessive saturation
- Color bleeding
- Colored graphite
- Colored shadows

---

# Success Criteria

The color system succeeds when

- Every knowledge region is instantly recognizable.
- Structural materials remain visually neutral.
- Electrical activity communicates information.
- Multiple active regions remain visually distinguishable.
- The overall scene feels premium rather than flashy.

This document serves as the authoritative color reference for Project Cortex.
