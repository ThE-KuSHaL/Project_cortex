# Project Cortex
# 03_references/reference_ui.md

Version: 1.0

Purpose

This document defines the visual language and behavior of every interface element surrounding the Engineer's Brain.

The UI exists to support the brain.

The UI is never the centerpiece.

---

# UI Philosophy

The interface should feel like precision instrumentation.

Not a dashboard.

Not an operating system.

Not a game HUD.

The user should feel like they are inspecting an engineered artifact.

---

# Primary Objective

The UI has only three responsibilities.

- Explain the brain.
- Allow interaction.
- Stay visually secondary.

If the UI attracts more attention than the brain, it has failed.

# Knowledge Panels

Every knowledge family owns one panel.

Panels should feel like floating technical instruments.

Characteristics

- Glassmorphism
- Soft transparency
- Thin illuminated border
- Rounded corners
- Soft shadow
- Backdrop blur
- Minimal visual noise

---

# Panel Placement

The brain always remains visible.

Panels never overlap the brain.

---

# Panel Behaviour

Panels deploy.

They never pop.

They never bounce.

Deployment Sequence

Connector grows

↓

Energy packet travels

↓

Panel fades in

↓

Panel rises slightly

↓

Content appears

Reverse the sequence when closing.

---

# Panel Contents

Each panel contains

- Knowledge Family
- Short Description
- Technologies
- Tools
- Related Projects (Future)
- Experience Level (Optional)
- External Links (Future)

Content should remain concise.

---

# Typography

Typography supports the visualization.

Never dominates it.

Characteristics

- Clean
- Modern
- Geometric
- High readability
- Consistent spacing

Avoid decorative fonts.

---

# Icons

Icons should remain minimal.

Use only when they improve recognition.

Avoid excessive icon usage.

---

# Connector Lines

Connector lines represent knowledge pathways.

Characteristics

- Thin
- Precise
- Slight glow
- Animated
- Direct

Avoid unnecessary curves.

Avoid tangled routing.

---

# Connector Nodes

Every connector begins at an illuminated node.

Node states

Idle

Hover

Selected

Nodes should communicate interaction availability.

---

# Knowledge Labels

Every lobe contains a label.

The label should

- Remain readable
- Face the camera
- Scale appropriately
- Never clip into geometry

Hovering the label should behave identically to hovering the lobe.

---

# Hover Behaviour

Hovering

- Brightens the lobe
- Highlights the label
- Increases packet activity
- Illuminates connector node

Panels do not appear on hover.

---

# Selection Behaviour

Clicking

- Activates region
- Deploys panel
- Maintains highlight
- Keeps connector active

Clicking again deselects.

---

# Multi-Selection

Multiple regions may remain active simultaneously.

Panels should arrange automatically.

Never overlap.

Never hide the brain.

Maintain balanced spacing.

# Skill Chips

Technologies should appear as compact chips.

Characteristics

- Rounded
- Minimal
- Slight transparency
- Soft border

Avoid oversized badges.

---

# Responsive Behaviour

Desktop

Panels surround the brain.

Tablet

Panels occupy left and right columns.

Mobile

Panels appear below the brain.

The interaction model remains identical.

---

# Background UI

The background may include

- Very subtle grid
- Technical markings
- Small geometric accents

Never distract from the brain.

---

# Future UI Expansion

Reserved areas

- Project previews
- GitHub integration
- Timeline
- Patent links
- Research papers
- Live statistics

The layout should allow future growth.

---

# Common Mistakes

Avoid

- Large opaque cards
- Bright backgrounds
- Thick borders
- Excessive animations
- Oversized text
- Decorative elements
- Panels covering the brain
- Random UI placement

---

# Success Criteria

The UI succeeds when

- The brain remains the visual centerpiece.
- Every interaction feels intentional.
- Information is easy to discover.
- Multiple panels remain organized.
- The interface enhances the experience without becoming the experience.

This document serves as the authoritative UI reference for Project Cortex.
