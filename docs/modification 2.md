# Purpose

This document contains the second visual refinement pass for Project Cortex.

The objective is to elevate the Engineer's Brain from a technically impressive interactive component into a premium portfolio centerpiece through improvements to the presentation platform, environmental detail, circuit generation, shader polish and interaction feedback.

No architectural redesigns are permitted.

Only visual refinement.

---

# Design Philosophy

The visitor should immediately think

> "This is an engineered artifact."

—not—

> "This is a 3D brain with glowing lines."

Every improvement should reinforce the illusion that this object was manufactured rather than modeled.

---

# 1. Presentation Platform Redesign

Current State

Simple illuminated circular platform.

Desired State

The platform should become an advanced engineering pedestal.

Imagine

- precision CNC-machined metal
- laboratory-grade display stand
- Stark Industries prototype showcase
- futuristic semiconductor inspection platform

The platform should feel expensive.

---

# Platform Construction

The platform should consist of multiple layers.

Layer 01

Structural Base

Dark graphite metal.

Very rough.

Heavy.

---

Layer 02

Primary Ring

Brushed titanium.

Machined grooves.

Engraved radial cuts.

---

Layer 03

Circuit Layer

Fine PCB routing engraved into the metal.

Tiny illuminated vias.

Microscopic copper traces.

No large empty surfaces.

---

Layer 04

Energy Ring

A recessed circular light channel.

Soft cyan glow.

Slow pulsing.

No harsh bloom.

---

Layer 05

Calibration Ring

Thin holographic markings.

Small engineering ticks.

Reference markings.

Tiny coordinate labels.

Subtle.

Readable only when zoomed.

# Surface Detail

Every visible centimeter of the platform should contain detail.

Examples

- etched grooves
- machining marks
- micro screws
- bolt heads
- recessed panels
- PCB routing
- inspection ports
- tiny warning markings
- maintenance labels
- serial identifiers

The viewer should continuously discover new detail while zooming.

# PCB Pattern Generation

Current

Decorative glowing lines.

Desired

Procedurally generated PCB layout.

The routing should resemble a motherboard.

Characteristics

- hierarchical buses
- branch routing
- local routing
- micro routing
- routing islands
- pads
- vias
- chip footprints

Avoid random noise.

---

# Routing Algorithm

Replace decorative routing with engineered routing.

Preferred characteristics

- mostly 45° turns

- occasional 90° intersections

- constant trace widths

- proper spacing

- parallel routing

- branching hierarchy

- believable manufacturing logic

Routing should appear generated by an electronic design automation (EDA) tool rather than artistic noise.

---

# Circuit Density

Increase routing density significantly.

Target

Approximately 80–90% surface coverage.

Very little unused substrate.

The surface should appear computationally dense.

---

# Via System

Introduce thousands of tiny vias.

Characteristics

- circular
- metallic
- recessed
- slightly emissive

Vias should naturally interrupt routing and reinforce manufacturing realism.

---

# Chip Population

Populate the surface with tiny electronic components.

Examples

- IC packages
- QFN packages
- BGA footprints
- passive arrays
- capacitors
- resistors
- logic packages
- processor outlines

These should remain subtle.

The goal is visual richness.

# Platform Animation

The platform itself should feel alive.

Introduce

- slow concentric energy pulses
- moving calibration sweep
- tiny blinking status LEDs
- subtle rotating holographic indicators
- periodic energy synchronization

Movement should remain slow and elegant.

---

# Brain Mount

The brain should not appear to float.

Introduce a proper support assembly.

Suggestions

- carbon fiber column
- titanium support
- illuminated power conduit
- segmented mechanical neck

The mount should visually connect the platform to the brain.

---

# Brain Synchronization

Every platform pulse should subtly synchronize with the brain.

Examples

- energy pulse reaches the base
- power travels upward
- packets propagate into the brain
- idle glow responds

The platform should feel like the power source.

---

# Hover Behaviour Revision

Current

Hover acceleration begins strongly then fades despite continuous hover.

Required

Hover state remains active while the pointer remains inside the region.

No automatic timeout.

---

Hover Effects

Maintain

- increased packet speed
- increased emissive
- brighter routing
- stronger bloom
- brighter rim lighting

until pointer exit.

---

# Hover Speed

Reduce hover packet acceleration by approximately 50%.

The current transition feels too aggressive.

Desired behaviour

Confident.

Controlled.

Intentional.

- branching packets
- convergence
- hotspot bursts
- packet priority lanes
- idle diagnostics

Avoid synchronized movement.

# Success Criteria

The Engineer's Brain should no longer appear to rest on a decorative platform.

Instead, it should appear mounted upon a precision-engineered computational pedestal whose circuitry, materials, energy systems and manufacturing detail convincingly suggest that the brain is an advanced technological artifact actively powered by the platform beneath it.

This modification document refines presentation only.

It does not alter the established architecture, interaction model or software systems defined in the original Project Cortex specifications.
